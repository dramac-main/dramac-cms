# DRAMAC CMS — Buttons & Interactive Components Master Plan

## Executive Vision

Transform DRAMAC's button and interactive component system from a functional but inconsistent set of 4 components into a **cohesive, conversion-optimised, world-class interactive element library** capable of producing click-worthy CTAs, accessible navigation actions, and engagement-driving UI elements that rival Stripe, Linear, Vercel, and Apple — all generated by the AI Designer with **zero human adjustment**.

Buttons are the **conversion engine of every website**. Every user journey ends with a click. Every CTA, every form submission, every navigation action flows through a button. If buttons are perfect — accessible, beautiful, fast, and consistent — conversion rates rise across every site the platform generates. This plan treats interactive components as the highest-ROI category in DRAMAC's component system.

## 3. Architecture Principles

### 3.1 Inline Styles for All Colours (No Hardcoded Tailwind)

```typescript
// ❌ WRONG — Hardcoded Tailwind colour, breaks dark mode
"bg-red-600 text-white hover:bg-red-700"

// ✅ CORRECT — Inline styles, theme-aware
style={{
  backgroundColor: backgroundColor || 'var(--color-destructive, #dc2626)',
  color: textColor || 'var(--color-destructive-foreground, #ffffff)',
}}
```

**Rule:** Every colour must be applied via `style={{}}` with a CSS variable fallback. Tailwind classes are ONLY for structural properties (padding, flex, grid, transitions).

### 3.2 Composition Over Duplication

```typescript
// ❌ CTA re-implements button rendering logic
<a className={`px-6 py-3 ${buttonColorClasses}...`}>Get Started</a>

// ✅ CTA composes ButtonRender
<ButtonRender
  label={buttonText}
  href={buttonLink}
  variant={buttonStyle === 'outline' ? 'outline' : 'primary'}
  size={buttonSize}
  backgroundColor={buttonColor}
  textColor={primaryButtonTextColor}
  ...
/>
```

### 3.3 Icon Name System for AI

```typescript
// ❌ Current — AI can't specify icons (requires ReactNode)
iconLeft={<ArrowRight className="w-4 h-4" />}

// ✅ Target — AI specifies icon by name, render resolves
iconName="arrow-right"  // Resolved from Lucide icon library at render time
iconPosition="right"
```

### 3.4 CSS Variables as Design Tokens

```css
/* Button design tokens — injected by renderer.tsx */
--btn-radius: 0.5rem;
--btn-font-weight: 500;
--btn-transition: 200ms;
--btn-focus-ring: var(--color-primary);
--btn-focus-ring-width: 2px;
--btn-focus-ring-offset: 2px;

/* Variant colours — theme-aware */
--color-primary: #3b82f6;
--color-primary-foreground: #ffffff;
--color-secondary: rgba(107, 114, 128, 0.12);
--color-secondary-foreground: inherit;
--color-destructive: #dc2626;
--color-destructive-foreground: #ffffff;
--color-success: #16a34a;
--color-success-foreground: #ffffff;
--color-warning: #d97706;
--color-warning-foreground: #ffffff;
```

### 3.5 Smart Defaults by Context

```typescript
// AI Designer rules:
// - Hero section → Button size "lg", fullWidthMobile, hoverEffect "lift"
// - Card footer → Button size "sm", variant "outline"
// - Form submit → Button type "submit", loading states enabled
// - Navigation → Button variant "ghost", size "sm"
// - Footer → Button variant "link", no padding
```

### 3.6 Accessibility by Default

Every interactive element MUST:

- Have visible focus indicator (`:focus-visible` ring, not `:focus`)
- Support keyboard activation (Enter + Space)
- Use semantic HTML (`<button>` or `<a>`, never `<div onClick>`)
- Have accessible name (label text or `aria-label`)
- Convey state changes to screen readers (`aria-disabled`, `aria-busy`)
- Meet WCAG 2.1 AA contrast (4.5:1 text on button, 3:1 button on background)

### 10.3 AI Quality Scoring for Buttons

```typescript
function scoreButtonQuality(page: GeneratedPage): number {
  let score = 100;
  const issues: string[] = [];

  // Rule: Hero must have at least 1 button
  if (page.heroSection && page.heroSection.buttons.length === 0) {
    score -= 20;
    issues.push("Hero section has no call-to-action button");
  }

  // Rule: Maximum 2 buttons per section (primary + secondary)
  page.sections.forEach((section) => {
    if (section.buttons.length > 2) {
      score -= 10;
      issues.push(
        `Section "${section.title}" has ${section.buttons.length} buttons — max 2`,
      );
    }
  });

  // Rule: Button text should be actionable (verb + noun)
  page.allButtons.forEach((btn) => {
    if (["Click Here", "Submit", "Button", "Link"].includes(btn.label)) {
      score -= 5;
      issues.push(
        `Button "${btn.label}" is generic — use action verb + benefit`,
      );
    }
  });

  // Rule: Contrast check
  page.allButtons.forEach((btn) => {
    const contrast = getContrastRatio(btn.textColor, btn.backgroundColor);
    if (contrast < 4.5) {
      score -= 15;
      issues.push(
        `Button "${btn.label}" fails contrast: ${contrast.toFixed(1)}:1 (need 4.5:1)`,
      );
    }
  });

  return { score: Math.max(0, score), issues };
}
```

### 12.1 Icon Resolution Performance

```typescript
// ❌ WRONG — Import entire icon library
import * as LucideIcons from "lucide-react";

// ✅ CORRECT — Dynamic import with caching
const iconCache = new Map<string, React.ComponentType>();

async function resolveIcon(name: string): Promise<React.ComponentType | null> {
  if (iconCache.has(name)) return iconCache.get(name)!;

  try {
    const mod = await import(`lucide-react/dist/esm/icons/${name}.js`);
    iconCache.set(name, mod.default);
    return mod.default;
  } catch {
    console.warn(`[Button] Icon "${name}" not found in Lucide library`);
    return null;
  }
}
```

## Key Rules Summary

### Button Rules

1. **NEVER hardcode Tailwind colour classes** — all colours via inline styles + CSS variables
2. **ALWAYS render `<button>` or `<a>`** — never `<div onClick>`
3. **href → `<a>`, no href → `<button>`** — no exceptions
4. **Max 2 buttons per section** — 1 primary + 1 secondary
5. **Focus ring via `:focus-visible`** — not `:focus` (avoids ring on click)
6. **44px minimum touch target** on mobile
7. **aria-busy for loading, aria-disabled for disabled** — on top of native attributes
8. **Reduced motion respected** — all animations disabled when `prefers-reduced-motion`

### Badge Rules

9. **Every badge colour from CSS variable** — no hardcoded hex
10. **`color-mix()` for variant backgrounds** — creates consistent tinted backgrounds from semantic colours

### Social Links Rules

11. **15+ platforms supported** — keep list extensible
12. **Brand colours are opt-in** — `useBrandColors` flag, default is monochrome
13. **Aria-labels on every icon** — platform name announced by screen reader

### CTA Rules

14. **Compose ButtonRender** — don't duplicate button rendering logic
15. **Structured trust badges** — `{ image, alt, link }`, not bare URLs
16. **SSR-safe countdown** — useEffect-based, no hydration mismatch

### AI Integration Rules

17. **Button text is specific** — "Start Free Trial" not "Click Here"
18. **Context-appropriate variants** — hero=lg solid, card=sm outline, nav=ghost
19. **Industry-specific styling** — luxury=uppercase no-radius, tech=rounded medium
20. **Quality score ≥ 80** for any page's button usage
